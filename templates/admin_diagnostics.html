Service Health

This panel tells you whether the background jobs are busy or idle right now.

  • MSRP Diff shows the price sync job state.
  • Specs Backfill shows the item spec backfill state.
  • The log line counts help confirm that work is actually happening.
Runtime Snapshot

These are the app and container details that tend to change between deployments.

  • Use the app version and git SHA to confirm the image you expected is running.
  • UID/GID and PUID/PGID help catch permission problems in mounted volumes.
  • The PID 1 command is useful when debugging container entrypoints.
Schema Migrations

This history shows structural database migrations that have been applied.

  • The current version tells you what schema shape the database should have.
  • Each history entry is a numbered table change, applied once in order.
  • If this lags behind code, startup may still need to finish a migration step.
Bootstrap History

Bootstrap history tracks one-time startup data fixes and seed work.

  • These steps seed defaults, clean bad data, and normalize the collection.
  • This is separate from schema migrations because it changes data behavior, not tables.
  • It helps explain why startup may touch the database even when the schema is already current.
Storage & Paths

This section tells you whether the app can write where it expects to write.

  • Read-only data or log directories usually mean a bind mount or permission issue.
  • The SQLite file check appears only when SQLite is the active backend.
  • Use these checks first when uploads, logging, or startup state looks odd.
